Empezemos por conocer que es Git, este es un es sistema de control de versiones distribuido, nace entonces Github un hosting para git, gratuito para proyectos de característica opensource
Primero debemos configurar el entorno local para trabajar bajo github luego sabremos cómo realizar la primera subida de un proyecto al repositorio.

 

Ahora instalaremos el git y procedemos a configurarlo.
Ahora descargaremos una empaquetación del git listo para instalar, desde, de la siguiente ubicación: http://git-scm.com/
Esta instalación es sencilla, tras la misma comprobaremos la versión instalada ejecutando por línea de comandos:


1

git –versión

Seguidamente configuraramos el usuario, e-mail y los decoradores de la salida por consola, se siguen los siguientes pasos para la configuración de git.
Agregaremos a la configuración dada la creación de un archivo de exclusiones. Es normal que por el sistema operativo o por el sistema de gestión del ciclo de vida del proyecto, se generen en el directorio de nuestra aplicación archivos que no queremos que formen parte del repositorio. Para esto se puede tener una configuración general por proyecto o una global para todos los proyectos.
El resultado de todas estas configuraciones debería ser un archivo .gitconfig en nuestro directorio de usuario que contenga:

GitHub usa una clave tipo ssh para establecer una conexión segura entre el usuario y el servidor de git. Debemos generar una calve y añadir la misma a nuestro perfil GitHub.
Para generar una nueva clave, desde el terminal se deben ejecutar los siguientes comandos:


1

ssh-keygen -t rsa -C "your_email@yourcompany.com"

Enseguida nos solicitará una ubicación para la clave, por defecto nos ofrece la siguiente ruta: /Users/your_user_directory/.ssh/id_rsa y una "frase cualquiera" como contraseña.

Subir el proyecto
Una vez tenemos el entorno configurado y montado ya podemos subir el proyecto para compartirlo.
Al tratarse de un nuevo proyecto lo primero es ubicarnos a nivel de la carpeta padre del proyecto (allí donde se encuentra, por ejemplo, el directorio .metadata) y ejecutar:

1

git init

Con ello crearemos el repositorio de git, que es el mismo directorio .git en la misma ubicación (si tuvieramos que volver a crearlo basta con borrarlo).
A continuación, agregamos todos los fuentes al repositorio, con comando:


1

git add *

Para revisar lo agregado al repositorio podemos ejecutar el comando:


1

git status

Todo lo que se liste será lo que se suba con el proyecto.
Lo último es hacer un commit, asignar al repositorio remoto y hacer un push


1
2
3

git commit -m 'Subo la estructura del proyecto al repositorio de GitHub'
git remote add origin git@github.com:my_company/my_project.git
git push -u origin master

Ahora en el sitio del proyecto en GitHub se tendra el push
